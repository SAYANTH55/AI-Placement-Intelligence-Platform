# Contribution Guidelines

1. **Branching**: Use feature branches (`feature/your-task`).
2. **Pulls**: Always pull the latest `main` before starting work.
3. **Reviews**: Submit Pull Requests (PRs) for review by team members.
4. **Commits**: Use descriptive commit messages.

## Development Workflow

1. Update your local repository.
2. Create a new branch.
3. Implement changes and add tests.
4. Push and create a PR.
